Les adaptateurs convertissent la sortie du support de référence en JSON standardisé,
au format Bencher Metric (BMF).
Les adaptateurs sont exécutés sur le serveur API lorsqu'un nouveau rapport est reçu.
Voir [l'aperçu du benchmarking][benchmarking overview] pour une explication plus approfondie.
Ils peuvent être spécifiés pour [la sous-commande CLI `bencher run`][bencher run]
avec l'option `--adapter`.
Si aucun adaptateur n'est spécifié, [l’adaptateur `magic`][magic adapter] est utilisé par défaut.

Il est préférable d'utiliser l'adaptateur le plus spécifique à votre cas d'utilisation.
Cela fournira à la fois le parsing le plus précis et le plus performant.
Par exemple, si vous analysez la sortie du bench [libtest de Rust][rust libtest],
vous devriez utiliser l'adaptateur `rust_bench`, et non l'adaptateur `magic`.
Voir notre
[page Bencher perf][perf page] pour une bonne comparaison.
