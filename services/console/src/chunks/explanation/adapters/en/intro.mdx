Adapters convert benchmark harness output into standardized JSON,
Bencher Metric Format (BMF).
The adapters run on the API server when a new report is received.
See the [benchmarking overview][benchmarking overview] for a more in-depth explanation.
They can be specified for [the `bencher run` CLI subcommand][bencher run]
with the `--adapter` option.
If no adapter is specified, [the `magic` adapter][magic adapter] is used by default.

It is best to use the most specific adapter for your use case.
This will provide both the most accurate and performant parsing.
For example if you are parsing Rust [libtest bench][rust libtest] output,
you should use the `rust_bench` adapter, and not the `magic` adapter.
See our
[Bencher perf page][perf page] for a good comparison.
