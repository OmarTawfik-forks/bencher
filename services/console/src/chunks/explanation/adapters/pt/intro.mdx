Adaptadores convertem a saída do harness de benchmark para JSON padronizado,
Formato de Métrica do Bencher (BMF).
Os adaptadores são executados no servidor API quando um novo relatório é recebido.
Veja a [visão geral de benchmarking][visão geral de benchmarking] para uma explicação mais aprofundada.
Eles podem ser especificados para o subcomando CLI [o `bencher run`][bencher run]
com a opção `--adapter`.
Se nenhum adaptador for especificado, [o adaptador `magic`][adaptador magic] é usado por padrão.

É melhor usar o adaptador mais específico para o seu caso de uso.
Isso proporcionará a análise mais precisa e performática.
Por exemplo, se você estiver analisando a saída do bench [libtest do Rust][rust libtest],
você deve usar o adaptador `rust_bench`, e não o adaptador `magic`.
Veja nossa
[página de desempenho do Bencher][página perf] para uma boa comparação.
