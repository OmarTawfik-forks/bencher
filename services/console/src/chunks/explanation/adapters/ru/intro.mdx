Адаптеры преобразуют вывод оболочек для бенчмарков в стандартизированный JSON,
Формат Метрики Бенчмарка (BMF).
Адаптеры запускаются на сервере API при получении нового отчёта.
Смотрите [обзор бенчмарков][benchmarking overview] для более глубокого объяснения.
Их можно указать для подкоманды CLI [bencher run][bencher run]
с помощью опции `--adapter`.
Если адаптер не указан, по умолчанию используется [адаптер `magic`][magic adapter].

Лучше всего использовать наиболее конкретный адаптер для вашего случая.
Это обеспечит наиболее точный и производительный разбор.
Например, если вы анализируете вывод [libtest bench][rust libtest] для Rust,
вы должны использовать адаптер `rust_bench`, а не адаптер `magic`.
Смотрите нашу
[страницу Bencher perf][perf page] для хорошего сравнения.
