Adapter konvertieren die Ausgabe von Benchmark-Harness in das standardisierte JSON,
Bencher Metric Format (BMF).
Die Adapter werden auf dem API-Server ausgeführt, wenn ein neuer Bericht empfangen wird.
Siehe die [Übersicht über das Benchmarking][benchmarking overview] für eine ausführlichere Erklärung.
Sie können für das [CLI-Unterkommando `bencher run`][bencher run]
mit der Option `--adapter` angegeben werden.
Wenn kein Adapter angegeben ist, wird standardmäßig [der `magic` Adapter][magic adapter] verwendet.

Es ist am besten, den spezifischsten Adapter für Ihren Anwendungsfall zu verwenden.
Dies stellt sowohl die genaueste als auch leistungsstärkste Analyse zur Verfügung.
Wenn Sie zum Beispiel Rust [libtest bench][rust libtest]-Ausgabe parsen,
sollten Sie den `rust_bench` Adapter und nicht den `magic` Adapter verwenden.
Sehen Sie auf unserer
[Bencher Perf-Seite][perf page] einen guten Vergleich.
