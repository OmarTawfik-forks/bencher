## ベンチマークコマンド

最初の引数`bencher run`はオプションのベンチマークコマンドです。
これは、あなたのベンチマークハーネスを起動するコマンドです。
これは`BENCHER_CMD`環境変数を使って設定することもできます。
デフォルトでは、このコマンドはシェルで実行され、これは`--shell`と`--flag`オプションで設定できます。
その出力は、ベンチマークハーネスアダプタにより解析され、これは`--adapter`オプションを使用して設定できます。
ただし、ベンチマークハーネスがファイルに出力する場合、出力ファイルパスを指定するために、`--file`オプションも使用する必要があります。

コマンドをシェルで実行しないで済む場合、`--exec`フラグを使用するか、コマンドに追加の引数として追加の引数を提供することで`bencher run`ができます。

シェル形式：
```bash
bencher run "bencher mock"
```

実行形式：
```bash
bencher run bencher mock
```

ベンチマークコマンドは、`--iter`オプションを使用して複数回実行でき、
これらの結果は`--fold`オプションを使って1つの結果に統合できます。
反復のいずれかが失敗すると、全体のコマンドは`--allow-failure`フラグが設定されている場合を除き、失敗したと見なされます。

ベンチマークコマンドが指定されていないが`--file`オプションが設定されている場合、`bencher run`は代わりに出力ファイルパスから読み込みます。
ベンチマークコマンドと`--file`オプションのどちらも指定されていない場合、`bencher run`は代わりに`stdin`から読み取ります。
これにより、別のコマンドの出力をファイルに保存したり、それを`bencher run`にパイプで送信することができます。
