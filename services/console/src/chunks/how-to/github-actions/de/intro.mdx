Je nach deinem Anwendungsfall kannst du [durchgehende Benchmarks][continuous benchmarking] in GitHub Actions für deine:

- [Basis-Branch][base branch]
- [Pull-Anfragen][pull requests]
- [Pull-Anfragen aus Forks][fork pull requests]
    - ⛑️ Sicherer: [Benchmark Fork-PR und Upload vom Standard-Branch][benchmark fork pr and upload from default branch]
    - ⚠️ Riskanter: [Benchmark Fork-PR vom Ziel-Branch mit erforderlichen Reviewern][benchmark fork pr from target branch with required reviewers]

Stelle sicher, dass du [einen API-Token erstellt hast][create an api token]
und setze ihn als **Repository**-Geheimnis mit dem Namen `BENCHER_API_TOKEN`, bevor du fortfährst!
Navigiere zu `Dein Repo -> Einstellungen -> Geheimnisse und Variablen -> Aktionen -> Neues Repository-Geheimnis`.
Benenne das Geheimnis `BENCHER_API_TOKEN` und setze den Geheimniswert auf deinen API-Token.

In GitHub Actions,
[Geheimnisse werden dem Runner nicht übergeben, wenn ein Workflow von einem geforkten Repository ausgelöst wird][github actions secrets].
Daher müssen Sie einen Branch aus demselben Repository verwenden,
wenn Sie einen der unten stehenden Workflows mit einer Pull-Anfrage zu Ihrem Repository hinzufügen.
Wenn Sie versuchen, Bencher mit einer Pull-Anfrage von einem Fork hinzuzufügen,
wird das `BENCHER_API_TOKEN` Geheimnis nicht verfügbar sein.
`${{ secrets.BENCHER_API_TOKEN }}` wird eine leere Zeichenkette sein.
