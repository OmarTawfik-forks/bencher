## Comparación de Rendimiento de Diesel

El [primer commit][github diesel first commit] de Diesel fue realizado por el creador del proyecto [Sage Griffin][github sgrif] el 23 de agosto de 2015.
Esto fue solo tres meses después de que se [lanzara Rust 1.0][rust v1],
por lo que originalmente usaron el arnés de pruebas de rendimiento integrado [libtest bench][libtest bench].
Luego, [el 03 de noviembre de 2020][github diesel 2507], cambiaron a usar [Criterion][criterion] como su arnés de pruebas de rendimiento.

Cuando Diesel cambió a usar Criterion, también empezaron a [compararse con otros ORMs de Rust][github diesel bench].
Esta suite de comparación de benchmarks ahora incluye:

- [Diesel][github diesel]
- [SQLx][github sqlx]
- [Rustorm][github rustorm]
- [Quaint][github quaint]
- [Postgres][github postgres]
- [Rusqlite][github rusqlite]
- [Mysql][github mysql]
- [diesel-async][github diesel async]
- [wtx][github wtx]
- [SeaORM][github seaorm]

No fue hasta el 05 de mayo de 2021 que el contribuidor y ahora mantenedor principal
[Georg Semmler][github weiznich] comenzó a rastrear los resultados de los benchmarks de Diesel a lo largo del tiempo.
Él creó [una GitHub Action nocturna][github actions diesel metrics] para ejecutar los benchmarks
y [un repositorio de métricas separado][github diesel metrics] para rastrear los resultados de los benchmarks.
Debido al ruido de los runners de GitHub Action, estos resultados solo se consideran para mostrar una tendencia general en cambios grandes.
Tampoco hay monitoreo configurado, por lo que los mantenedores de Diesel tienen que verificar manualmente los resultados para detectar una regresión de rendimiento.

Para obtener una mejor comprensión del impacto del rendimiento de un cambio,
[el 20 de marzo de 2022][github diesel 3103]
se añadió la realización de benchmarks basados en el conteo de instrucciones [usando `criterion-perf-events`][github criterion perf events].
Debido a que `criterion-perf-events` utiliza eventos `perf` de Linux, no puede ejecutarse en GitHub Actions.
Por lo tanto, estos benchmarks deben ejecutarse localmente.
